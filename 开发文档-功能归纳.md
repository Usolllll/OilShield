# 企业油价风险管理平台功能方案

## 一、平台功能架构与用户流程

本章节从功能分布、核心逻辑以及用户交互的角度，对平台进行详细设计。平台遵循"结论先行、逐层下钻"原则，满足管理层快速决策与执行层精细分析的不同需求。

> **总体为五个分区**

### 1.1 平台整体功能分区

平台划分为五大核心功能区：

1. **风险总览仪表盘 (Dashboard)**：管理层快速判断风险状态的入口。
2. **企业油价风险健康度诊断模块**：构建企业专属风险画像，量化脆弱性。
3. **油价预测与多情景分析模块**：提供概率化的未来价格路径。
4. **对冲策略模拟与方案对比模块**：输出可执行的套期保值建议。
5. **AI 风险日报与报告中心**：提供持续的信息追踪与 AI 智能解读。

### 1.2 首页 / 风险总览仪表盘（管理层入口）

**功能定位**：让管理层在 1 分钟内判断"当前是否存在显著油价风险"。展示经过模型整合后的核心结论：

- **价格现状**：当前油价水平与当日变动情况。
- **未来预期**：未来一年油价概率区间可视化路径（阴影图/路径图）。

### 1.3 企业油价风险健康度诊断（平台核心特色）

**功能定位**：将外部油价变化转化为"与企业高度相关的风险指标"。

#### (1) 信息采集机制（分层递进，默认存储）

通过低门槛、分阶段的输入，构建企业油价风险画像：

- **基础层（低门槛）**：
  - 行业类型（航空 / 化工 / 制造 / 物流等）
  - 经营区域（国家 / 区域）
  - 油品类型（原油 / 成品油 / 航空燃油）

- **进阶层（风险量化）**：
  - 年 / 月油品消耗量（或区间）
  - 油品成本占总成本比例
  - 预算油价
  - 主要结算币种

- **经营特征层（精细化建模）**：
  - 成本是否可向下游传导
  - 对油价波动的容忍度（低 / 中 / 高）
  - 支持企业上传财务报表，平台自动解析相关字段，并由客户确认

#### (2) 风险健康度输出

该模块评估在当前价格环境下企业的脆弱性：

- **健康度评分**：$0-100$ 分的综合评分。
- **风险来源拆解**：解释分数低的原因（如：成本传导性差、无对冲头寸、价格处于历史高位）。
- **核心指标**：油价—成本弹性系数、不同情景下的利润与现金流分布。
- **风险预警**：企业当前油价风险健康等级（低/中/高）。

### 1.4 油价预测与多情景分析（风险输入层）

**功能定位**：为风险管理提供"概率化的未来空间"。

- **基准路径**：概率最高、最有可能发生的路径。
- **风险路径**：高油价风险路径（例：$20\%$ 概率发生的高位运行）。
- **有利路径**：低油价情景。
- **极端压力情景**：模拟地缘政治、供应中断等小概率但高代价的事件。

**交互设计**：用户可在界面中切换不同情景，查看对应的驱动因素说明。

### 1.5 对冲策略模拟与方案对比（决策支持核心）

**功能定位**：唯一直接输出行动方案的模块，解决"该如何操作"的问题。

#### (1) 策略参数设置

用户设置对冲期限（3 个月 / 6 个月 / 1 年）、风险偏好（风险优先/成本优先/收益平衡）、工具偏好（期货/期权/组合）以及是否保留部分敞口。

#### (2) 策略输出与对比

平台基于优化模型，筛选 2–3 种代表性方案进行并排展示：

- **完全保值方案**：风险最低，锁定成本。
- **低成本部分保护方案**：支付少量成本，防范极端上涨。
- **灵活收益增强方案**：在保护下限的同时，保留油价下跌带来的额外收益。

### 1.6 AI 油价风险日报与报告中心

> **集成于个人信息**

- **AI 驱动**：每日推送油价变动核心驱动因素、行业专属风险提示及未来前瞻事件。
- **智能解读**：所有看板与报告支持 AI 点击解读功能。用户可框选特定指标或结论，AI 立即提供底层的逻辑解释与背景说明。
- **历史归档**：支持 PDF/PPT 一键导出，方便内部会议使用。

## 二、系统交互逻辑与数据流设计

### 2.1 与模型组的交互

#### 通过 API 与算法模型交互（请求响应报文）

当用户点击"开始预测"或"优化策略"时，后台发起 API 调用：
(例)
**请求报文 (Request JSON)**：

```json
{
  "request_id": "REQ12345",
  "enterprise_id": "ENT_001",
  "module": "hedge_optimization",
  "params": {
    "risk_preference": "risk_averse",
    "horizon": "6M",
    "exposure_amount": 10000,
    "tools": ["futures", "options"]
  }
}
```

**响应报文 (Response JSON)**：

```json
{
  "status": "success",
  "timestamp": "2023-10-27T10:00:00Z",
  "data": {
    "recommended_schemes": [
      { "name": "Full Protection", "hedge_ratio": 0.9, "expected_cost": 75.5 },
      { "name": "Cost Effective", "hedge_ratio": 0.5, "expected_cost": 72.1 }
    ],
    "risk_score": 85
  }
}
```

### 2.2 与数据分析组的交互

**数据分析组（算法组）**：

#### (1) 与数据库的交互（核心任务）

- **历史数据录入**：
  - 通过自动化脚本，定时从外部数据源（如路透、彭博）抓取全球主要原油标的的每日结算价。
  - 对原始价格进行清洗（去噪、补全、货币换算），并写入数据库的"历史油价表"。

- **预测数据回写**：
  - 算法模型定时运行，生成基准、风险、有利、极端四种情景下的未来一年预测价格序列。
  - 将这些带有概率分布的时间序列数据写入数据库的"预测结果表"，供开发组调用。
